## Bab 1 : Perkenalan Amazon Web Service (AWS)

### 1.1 Apa itu Amazon Web Service (AWS)?

Amazon Web Service atau AWS adalah salah satu layanan penyedia komputasi cloud yang telah hadir di seluruh dunia. AWS merupakan layanan komputasi awan yang sangat canggih dan komprehensif. 

AWS mwenawarkan berbagai macalm layanan untuk setiap kegunaan, seperti layanan dasar awan yaitu komputasi, penyimpanan, dan keamanan jaringanm hingga seperti blockchain, machine learning, atau AI (Artificial Intellegence).

### 1.2 Seperti apa Kerja AWS?

AWS bekerja dengan model yang banyak digunakan pada komputasi awan umumnya, yaitu Server-klien yang dapat mencukupi banyak kebutuhan. Server-klien? apa maksudnya?

Oke, server klien adalah salah satu seknario kerja dari banyak komputer saat ini, jadi terdapat 2 pihak yaitu pihak Swerver dan Klien. Server sendiri adalah komputer yang mampu melayani sebuah data yang didasari adanya permintaan dari Klien, komputer server mampu melayani dari banyak hal, salah satu contohnya Web Server yang dimana menyediakan layanan aplikasi delam bentuk Web.

Untuk klien sendiri adalah komputer yang memberi permintaan ke server dan menerima data atau sesuatu hal tersebut. Contoh dari klien adalah seperti menerima aplikasi berbasis web dengan browser sebagai layanannnya.

Pada AWS ini, bisa kita tau bahwa sangat powerfull untuk digunakan, bisa menjadi Web Server, Database Server, dan masih banyak lagi. Tetapi yang jelas, AWS cenderung lebih ke skenario Server-Client contohnya Web Server dengan AWS Elastic Compute Cloud (EC2) yang nanti akan di pelajari.

### 1.3 Komputasi Awan

Disini dibahas tentang lingkup awal yang berikaitan AWS yaitu komputasi awan atau disebut Cloud Computing. Namun, apa itu Cloud Computing? Cloud Computing adalah layanan komputasi awan yang dapat digunakan dan dihargai sesuai dengan pemakaian (pay-as-you-go). AWS disini menggenkan konsep Komputasi Awan yang kental, dengan didukung banyak hal. Ini hal penting yang ada di Cloud Computing :

- Penggunaan Sesuai Kebutuhan, besar atau kecil bisa di sesuaikan dengan apa yang dibutuhkan.
- Sumber Daya IT yang tersedia, dalam waktu kapanpun sumber daya IT akan diusahakan selalu ada.
- Melalui Internet, jalur pengaksesan sumber daya tersebut bisa diakses dimana saja, hanya cukup dengan koneksi internet.

### 1.4 Model Penerapan untuk Komputasi Awan 

- Cloud-based Deployement
  Aplikasi dibangun atau dimigrasikan sepenuhnya ke cloud. Infrastruktur dapat dikelola sendiri atau menggunakan layanan tingkat tinggi untuk mengurangi kebutuhan pengelolaan.
- On-premises Deployment
  Juga dikenal sebagai private cloud, sumber daya diterapkan pada data center pribadi menggunakan teknologi virtualisasi. Model ini meningkatkan kontrol dan pemanfaatan sumber daya lokal.
- Hybrid Deployment
  Menggabungkan cloud dan on-premises, di mana sumber daya cloud terhubung ke data center lokal. Cocok untuk aplikasi lama atau situasi dengan regulasi khusus terkait data lokal.

### 1.5 Manfaat Komputasi Awan 

- Ubah pengeluaran di muka menjadi pengeluaran variabel.
- Hentikan biaya pengelolaan dan pemeliharaan data center.
- Berhenti menebak kapasitas.
- Meningkatkan kecepatan dan ketangkasan.
- Mendunia dalam hitungan menit.
